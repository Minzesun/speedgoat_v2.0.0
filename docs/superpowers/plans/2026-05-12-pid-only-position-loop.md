# PID-Only Position Loop Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `position_command_6064` the only operator position reference and remove manual velocity-feedforward tuning from the real-time position loop.

**Architecture:** The generated model keeps `PT-5 Position Loop` as the position controller, but its command path becomes PID-only: position error drives the PID state update, the resulting velocity command is saturated by `max_tracking_speed`, and the final command goes through the existing one-sample delay into the startup controller. `position_ff_velocity_60ff` remains as a reserved observability output fixed at zero so the signal surface stays familiar while `SGV2_POSITION_RATE_COMMAND_6064` and `position_rate_command_6064` are removed from the command interface.

**Tech Stack:** MATLAB R2021a, Simulink block generation scripts, Simulink Real-Time, MATLAB `functiontests`.
